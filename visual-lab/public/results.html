Известные ограничения · следующая итерация

  • Babel standalone в браузере — для production вынесите трансформацию в build-step (Vite/SWC). HTML/JSX готовы как референс.
  • Lighthouse + Playwright + axe-core — gates не выполнены здесь; собираются в CI на стороне репозитория.
  • Variant cap — три варианта вместо десяти из AGENT.md. Sankey / Hierarchical DAG / Knowledge Graph Explorer — следующие приоритеты (требуют d3-sankey / elkjs / sigma).
  • uses_later сейчас = cross-stage edges (8 шт). Если бизнес-логика "артефакт используется через 2-3 этапа" другая — компилировать поверх edges[] отдельным правилом.
  • Критпуть — эвристика type ∈ {process, result} AND coverage=full AND track=offbudget. В production должен приходить флагом из исходных JSON.
  • Sidebar nodes (process 1.0 в Stage 1) скрыты на канвасе — нуждаются в отдельном паттерне рендера в инспекторе или как chip-list.
